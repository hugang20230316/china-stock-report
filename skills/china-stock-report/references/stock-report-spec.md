# A股每日个股深度分析报告 — 生成规范

## 一、报告概述

- 报告名称：TOP6 值得关注个股深度汇总
- 输出格式：单个 HTML 文件（暗色主题，自包含样式和脚本）
- 文件命名：`stock_report_YYYYMMDD.html`（如 `stock_report_20260305.html`）
- 存放目录：`<workspace-root>/reports/`
- 保留策略：仅保留最近 5 份 HTML，生成新报告时自动清除多余旧文件
- **日期强校验**：报告文件名日期、标题日期、正文中标注的数据日期必须一致；例如 2026-03-09 生成的正式报告只能命名为 `stock_report_20260309.html`
- **最终产物强校验**：正式结果只能是 `stock_report_YYYYMMDD.html`，`.md`、`.json`、临时脚本输出仅可作为中间产物，禁止作为最终交付
- **数据新鲜度强校验**：正式报告中的行情、财务、K线截图必须明确标注抓取日期/时间；若无法确认是当日数据，则该字段不得写入正式报告

## 二、必须使用的工具链（强制！）

> **每次生成报告必须严格按以下工具链执行，禁止跳过或用其他方式替代。**

### 2.1 `/china-stock-analysis` Skill（股票筛选与分析）

- **用途**：股票筛选、个股深度分析、行业对比、估值计算
- **触发方式**：通过 Skill 工具调用 `china-stock-analysis`
- **必须使用的场景**：
  - 确定每日 TOP6 股票池（默认必须动态筛选；只有用户明确要求“按固定池生成”时，才允许验证固定池）
  - 生成每只股票的深度分析内容（核心逻辑、业绩亮点、估值判断、催化事件、风险提示）
  - 行业对比和估值合理性判断
- **输出**：结构化的分析数据，直接用于填充 HTML 模板
- **禁止**：不得自行编造分析内容，必须基于 skill 提供的数据和分析框架

### 2.2 `/figma` Skill（页面设计优化，可选）

- **用途**：读取 Figma 设计稿并据此优化 HTML 页面样式
- **触发方式**：通过 Skill 工具调用 `figma`
- **使用条件**：当有 Figma 设计稿 URL 时必须使用
- **适用场景**：
  - 首次生成报告模板时，如有 Figma 稿则以 Figma 为准
  - 页面样式大改版时
  - 用户提供新的 Figma 设计稿时
- **不适用**：日常生成报告（模板已固定时）仅需套用现有模板，无需每次调用

### 2.3 一体化数据拉取脚本 `fetch_data.js`（行情+财务）

- **用途**：一次调用并行获取 6 只股票的实时行情 + 财务数据（约 3-5 秒）
- **行情 API**：`https://push2delay.eastmoney.com/api/qt/stock/get`
- **财务 API**：`https://emweb.securities.eastmoney.com/PC_HSF10/NewFinanceAnalysis/ZYZBAjaxNew`
- **调用方式**：`node fetch_data.js --stocks '[{"code":"600089","market":"sh","name":"xxx"}, ...]'`
- **输出**：JSON 到 stdout，包含 quote（价格/涨幅/换手/市值/PE）和 finance（利润/营收/年份）
- **必须使用**：每次生成报告都要实时拉取，禁止使用缓存或历史数据
- **禁止用 subagent + WebFetch 替代**：直接用此脚本，避免 subagent 启动开销

### 2.4 Playwright 并行截图脚本 `screenshot.js`（K线截图）

- **用途**：自动截取东方财富行情页的日K线图（无需登录）
- **工具**：Playwright Node.js 脚本（`screenshot.js`），支持并行多 tab，默认 3 并发
- **调用方式**：`node screenshot.js --date YYYYMMDD --concurrency 3 --stocks '[...]'`
- **截图源**：东方财富行情页，URL 格式见 3.1
- **耗时**：6 只股票并行约 50-60 秒（串行需 120 秒+）
- **必须使用**：每次生成报告都要重新截图，确保K线图是最新的
- **登录/验证失败处理**：如果截图源需要登录或出现验证码，**立即停止并询问用户**，不要反复重试
- **失败重试**：单只超时自动重试 1 次

### 2.5 工具调用顺序 + 一步到位规则

**一步到位（绝对强制）**：用户说"整理一份今日股票推荐""生成今日股票报告"或任何触发本规范的请求时，必须自动执行完整流水线直到输出 HTML 文件，**禁止中途询问"是否要生成 HTML"**。

```
0. 复盘检查 → 扫描所有历史报告，对已满 5 个交易日且未复盘的逐一复盘（见 stock-summary-spec.md）
1. 读取教训 → 读取 stock-summary-spec.md「五、教训清单」，作为本次筛选的前置约束
2. /china-stock-analysis → 确定 TOP6 股票池 + 生成分析内容（筛选时逐条检查教训）
3. 东方财富 API + Playwright 截图 → 并行执行（用 subagent 或后台任务并行拉取财务数据和截图）
4. （可选）/figma → 如有 Figma 稿则按图优化页面
5. 组装 HTML → 套用模板 + 填充数据 + 嵌入截图
6. 保存 + 清理旧文件
```

**并行优化**：步骤 2 中，6 只股票的财务数据拉取和 K线截图是独立的，必须并行执行，不要串行等待。

### 2.7 日期与数据一致性规则（新增，绝对强制）

#### 2.7.1 先确认“今天是哪天”

- 开始生成前，必须先记录本机当前时间，示例：2026-03-09 10:10:24
- 报告头部必须写明：
  - 报告日期（如 2026-03-09）
  - 行情抓取时间
  - 财务数据抓取时间
  - K线截图时间
- **禁止凭印象使用“今天”“当天”**，必须用绝对日期

#### 2.7.2 禁止跨日混用数据

- **绝对禁止**把 2026-03-05 的旧价格、旧涨跌幅、旧 PE、旧截图回填到 2026-03-09 的报告里
- 旧报告、旧截图、旧缓存只允许用于：
  - 调试模板结构
  - 对比页面样式
  - 用户明确授权“允许历史数据回填”时
- 未经用户明确授权，**禁止**使用任何历史报告内容补正式报告字段

#### 2.7.3 当日字段写入条件

- 某字段只有在满足以下条件时，才允许写入正式 HTML：
  1. 能确认抓取成功
  2. 能确认字段对应日期为报告当天
  3. 能说明来源（东方财富截图 / 东方财富API / skill 分析结果）
- 任一条件不满足时：
  - 该字段留空或写“抓取失败”
  - 在报告页脚的“失败说明”中解释原因
  - **不得**用旧值代替

### 2.8 抓取失败处理规则（新增，绝对强制）

- /china-stock-analysis、东方财富 API、Playwright 任一关键步骤失败时：
  1. **立即停止继续拼装正式报告**
  2. 向用户汇报失败点、失败时间、失败接口/页面、错误信息
  3. 给出可选解决方案，由用户决定下一步
- **绝对禁止**：
  - 自行改用旧报告数据顶替
  - 自行删掉失败字段假装成功
  - 自行把 md、半成品、旧 HTML 当正式结果交付

#### 2.8.1 向用户汇报失败时的固定格式

```markdown
当天报告生成已中止。

- 报告日期：2026-03-09
- 失败步骤：[china-stock-analysis / 东方财富 API / Playwright 截图]
- 失败时间：[绝对时间]
- 错误信息：[原始错误或摘要]
- 影响范围：[哪几只股票、哪些字段受影响]

可选方案：
1. 继续重试当前数据源
2. 允许切换到你指定的新数据源
3. 仅输出成功抓到的当日字段，失败字段明确标注为空
```

### 2.9 耗时控制规则（新增，绝对强制）

- **5 分钟内**必须完成：
  - 日期确认
  - 数据源可用性验证（至少验证 1 只股票）
- **10 分钟内**必须给出第一次阶段性结果：
  - 要么开始批量生成
  - 要么明确说明哪个关键数据源失败
- **15 分钟内**若关键数据源仍失败：
  - 必须停止盲试
  - 必须询问用户处理方案
- **绝对禁止**在未向用户同步的情况下，长时间卡住 30-60 分钟

- 如果 `china-stock-analysis` 调用失败、返回空结果、字段日期不明确，必须立即停止并询问用户，禁止自行补名单、补理由、补结论

## 三、数据源

### 3.1 K线截图（东方财富，无需登录）

- 来源：东方财富行情页截图（K线图）
- K线图截取：仅截取日K线部分，不含其他面板
- 截图命名：`{序号}_{拼音名}_{代码}.jpeg`
- 截图工具：Playwright Node.js 脚本
- 东方财富股票页 URL 格式：
  - 沪市（60xxxx / 688xxx）：`https://quote.eastmoney.com/sh{code}.html`（code 为纯数字，如 601857）
  - 深市（00xxxx / 300xxx / 301xxx）：`https://quote.eastmoney.com/sz{code}.html`（code 为纯数字，如 300113）
- **禁止使用雪球**：雪球需要登录验证，容易超时，不再作为截图源
- 如果东方财富行情页也无法加载，立即停止并询问用户

### 3.2 财务数据（东方财富）

- API 地址：`https://emweb.securities.eastmoney.com/PC_HSF10/NewFinanceAnalysis/ZYZBAjaxNew`
- 参数：
  - `type=1`：年报数据
  - `type=0`：季报数据（取最新一期）
  - `code=SH600406` 或 `code=SZ300124`
- 提取字段：
  - `PARENTNETPROFIT`：归母净利润（转换为亿元，保留2位小数）
  - `TOTALOPERATEREVE`：营业总收入（转换为亿元，取整）
- 时间范围：最近4个完整年度 + 最新季报（如 2021-2024年报 + 2025Q3）

### 3.3 市场数据

每只股票需采集：
- 当日收盘价
- 今日涨幅（%）
- 换手率（%）
- 总市值（亿元）
- PE(动)（亏损时显示"亏损"）

## 四、股票筛选标准

### 4.1 筛选维度

| 维度 | 说明 |
|------|------|
| 行业趋势 | 所在行业处于上升周期或政策利好 |
| 业绩表现 | 营收/净利增长趋势，拐点信号 |
| 估值水平 | PE 合理性，与行业对比 |
| 资金面 | 换手率、主力资金流向 |
| 催化事件 | 近期政策、订单、技术突破等 |

### 4.1.1 涨停股过滤规则（强制）

- **尽量避免推荐当日涨停的股票**：涨停股次日追高风险大，不适合作为推荐标的
- 如果候选池中某只股票当日涨停（涨幅 ≥ 9.8% 或 ≥ 19.8%），应优先选择未涨停的同行业替代标的
- 只有在该股票具有极强的中长期逻辑且无合适替代时，才允许保留，但必须在风险提示中明确标注"当日涨停，追高风险较大"

### 4.1.2 近期涨幅过高过滤规则（强制）

- **禁止追高**：近 5 个交易日累计涨幅 ≥ 20% 或近 10 个交易日累计涨幅 ≥ 30% 的股票，直接排除
- 近 5 日涨幅 10%-20% 的股票，扣减 2 分并在风险提示中标注"短期涨幅较大，存在回调风险"
- 判断方式：通过东方财富行情 API 获取近期涨跌幅，或搜索该股票近期走势
- 优先选择处于低位启动、底部放量、或横盘突破初期的标的

### 4.2 推荐星级

| 星级 | 含义 |
|------|------|
| ★★★★★ | 强烈推荐：基本面优秀 + 趋势向上 + 估值合理 |
| ★★★★☆ | 推荐：逻辑清晰，有明确催化，可能估值偏高 |
| ★★★☆☆ | 关注：有亮点但风险较大，适合短线或左侧布局 |

### 4.3 投资周期标签

- `短期`：1-4周，事件驱动或技术形态
- `中期`：1-6个月，业绩兑现周期
- `长期`：6个月以上，产业趋势

## 五、HTML 页面结构

### 5.1 整体布局

```
标题 + 副标题（数据日期 + 数据来源）
├── 汇总表格（可展开行，按星级降序排列）
│   ├── 摘要行（点击展开/收起，手风琴模式）
│   │   └── 列：展开按钮 | 行业 | 股票名(链接) | 代码 | 股价 | 今日涨幅 | 换手率 | 总市值 | PE(动) | 星级 | 推荐理由
│   └── 详情行（展开后显示）
│       ├── 投资周期标签
│       ├── K线图 + 深度分析（左右布局，放上面）
│       ├── 柱状图：归母净利润 + 营业总收入（左右布局，放下面）
│       └── 风险提示
└── 页脚免责声明
```

### 5.2 交互行为

- 点击摘要行展开详情，再次点击收起
- 手风琴模式：同一时间只展开一行
- 展开按钮 `+` 旋转 45° 变为 `×` 表示已展开
- 股票名称可点击跳转东方财富对应页面（新标签页打开）
  - 沪市：`https://quote.eastmoney.com/sh{code}.html`
  - 深市：`https://quote.eastmoney.com/sz{code}.html`

### 6.2 字体规范

| 元素 | 字号 | 字重 |
|------|------|------|

| 表格数据 | 14px | normal |

| 分析正文 | 12px | normal |
| 风险提示 | 12px | normal |

### 6.3 柱状图规范

- 容器高度：160px
- 关键 CSS：`align-items:stretch` 让列撑满容器，`justify-content:flex-end` 让柱子底部对齐
- **柱子高度必须使用像素值（px），禁止使用百分比（%）**，百分比在 flex 布局中不可靠
- 计算公式：`柱高 = (当前值 / 最大值) * 120px`，最大值柱子固定 120px
- 正值柱：蓝色渐变 `#00d4ff` → `#0055aa`，向上生长
- 负值柱：红色渐变 `#cc2020` → `#ff4d4f`，向下生长
- 混合正负值时用虚线零轴分隔
- 柱子带发光阴影 `box-shadow`

### 6.4 表格规范

- 圆角 12px + 阴影
- 奇偶行交替背景
- hover 时蓝色边框高亮
- `border-collapse:separate`（支持圆角）

## 七、深度分析内容模板

每只股票的展开详情需包含以下分析模块（内容必须来自 `/china-stock-analysis` skill 的输出）：

### 7.1 核心逻辑
- 公司在行业中的地位
- 核心竞争力和护城河
- 主要增长驱动力

### 7.2 业绩亮点/拐点
- 最新财报关键数据及同比变化
- 营收/利润趋势判断
- 特殊事项说明（减值、一次性损益等）

### 7.3 估值与资金
- PE/PB 等估值指标 + 历史分位
- 换手率、市值等流动性指标
- 机构持仓或资金流向

### 7.4 关键催化
- 近期可能的催化事件
- 政策/行业/公司层面的利好

### 7.5 风险提示
- 业绩风险
- 估值风险
- 行业/政策风险

### 8.2 清理策略

每次生成新报告时执行：
1. 扫描目录下所有 `stock_report_*.html` 文件
2. 按文件名日期降序排列
3. 保留前 5 个，删除其余
4. 对应的 `assets/{date}/` 目录一并清理

伪代码：
```bash
# 获取所有报告文件，按名称倒序
files=$(ls -1r <workspace-root>/reports/stock_report_*.html)
# 跳过前5个，删除剩余
echo "$files" | tail -n +6 | xargs rm -f
```

## 九、定时任务集成建议

### 9.1 触发时间

- 建议：每个交易日收盘后（15:30 之后）
- 非交易日（周末、节假日）跳过

### 9.2 执行流程

```
1. 判断今日是否为交易日
2. 采集 TOP6 股票列表（可固定或动态筛选）
3. 抓取行情数据（价格、涨幅、换手率、市值、PE）
4. 抓取财务数据（东方财富 API）
5. 截取 K线图（Playwright）
6. 生成 HTML 报告
7. 保存到 `<workspace-root>/reports/stock_report_YYYYMMDD.html`
8. 清理超过 5 份的旧报告
9. （可选）发送通知
```

### 9.4 快速使用方式（新增）

#### 场景1：生成当天正式报告

用户可以直接说：

```text
按规范生成今天的股票报告，输出 stock_report_YYYYMMDD.html
```

系统必须执行：

1. 先确认当天绝对日期
2. 先验证 1 只股票的数据源是否可用
3. 调用 `china-stock-analysis` 做当天动态筛选或当天验证，不得直接套用 9.3 观察池
4. 验证通过后再批量抓取入选股票
5. 输出正式 HTML

#### 场景2：只看当天数据是否能抓到

用户可以直接说：

```text
先验证今天的数据源是否可用，只测 1 只股票
```

适用场景：

- 想先确认当天接口正常
- 不想一上来就跑完整流程
- 需要把总耗时控制在 5 分钟内

#### 场景3：抓取失败时怎么处理

用户可以直接说：

```text
如果当天抓取失败，不要回填旧数据，直接停下问我
```

此时系统必须：

- 立即停止
- 汇报失败原因
- 等用户指定解决方案
