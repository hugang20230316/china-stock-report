# A股推荐复盘与教训总结

## 一、概述

- 用途：对已满 5 个交易日且尚未复盘的历史推荐进行盈亏复盘，总结教训，指导后续筛选
- 规则源位置：`references/stock-summary-spec.md`
- 历史报告默认位于：`<workspace-root>/reports/`
- 本文件分两部分：**复盘规范**（固定）+ **历史教训**（持续积累）

## 二、复盘触发规则

### 2.1 自动触发（每次推荐前）

每次生成新的股票报告前，执行以下检查：

1. 扫描 `<workspace-root>/reports/stock_report_*.html` 所有历史报告
2. 对每份报告，计算其推荐日距今是否 **≥ 5 个交易日**
3. 检查该报告日期是否已出现在本文件「四、历史复盘记录」中（已复盘则跳过）
4. 对所有 **已满 5 个交易日 且 尚未复盘** 的报告，逐一执行复盘
5. 可能一次复盘多份报告（比如连续几天没生成新报告，积压了多份待复盘）
6. 全部复盘完成后，再开始新一轮筛选

### 2.2 手动触发

- 用户说"复盘""回顾推荐""检查盈亏"等关键词时触发

### 2.3 交易日计算

- 5 个交易日 ≈ 自然日 7 天（跳过周末和节假日）
- 例：03-05（周三）推荐 → 03-12（下周三）时已满 5 个交易日，可复盘
- 例：03-09（周一）推荐 → 03-16（下周一）时已满 5 个交易日，可复盘

## 三、复盘执行流程

### 3.1 读取历史推荐

1. 找到待复盘的报告文件：`<workspace-root>/reports/stock_report_YYYYMMDD.html`
2. 从 HTML 中提取每只股票的：
   - 股票名称、代码
   - 推荐日股价（收盘价）
   - 推荐星级
   - 推荐理由

### 3.2 获取当前行情

- 使用东方财富延迟行情 API 获取每只股票的当前/最新收盘价
- API：`https://push2delay.eastmoney.com/api/qt/stock/get`
- 参数：`fields=f43,f170&secid={市场代码}.{股票代码}`
  - 沪市 secid=1.{code}，深市 secid=0.{code}

### 3.3 计算盈亏

对每只股票计算：
- 涨跌幅 = (当前价 - 推荐日价) / 推荐日价 × 100%
- 盈亏状态：盈利 / 亏损 / 持平（±0.5% 内算持平）

### 3.4 分析亏损原因

对每只亏损股票，搜索分析：
- 期间是否有利空消息（业绩暴雷、政策打压、行业下行）
- 推荐时是否已处于高位（追高）
- 推荐时是否涨停（次日回调）
- 推荐逻辑是否已被证伪

### 3.5 提炼教训

从亏损案例中提炼可复用的筛选规则，格式：

```
【教训 #{编号}】{一句话总结}
- 来源：{日期} 推荐的 {股票名}（{代码}），亏损 {X}%
- 原因：{具体原因}
- 规则：{未来筛选时应遵守的规则}
```

### 3.6 输出复盘报告

在终端输出简要复盘结果，格式：

```
📊 复盘：{推荐日期} 推荐 → {复盘日期} 结算

| 股票 | 代码 | 推荐价 | 现价 | 涨跌幅 | 盈亏 |
|------|------|--------|------|--------|------|
| xxx  | xxx  | xx.xx  | xx.xx| +x.x%  | ✅盈利 |
| xxx  | xxx  | xx.xx  | xx.xx| -x.x%  | ❌亏损 |

胜率：X/6（XX%）
新增教训：#{编号} ...
```

## 六、筛选前检查规则

每次执行 `/china-stock-analysis` 筛选前，必须：

1. 读取本文件的「五、教训清单」
2. 对每只候选股逐条检查是否违反已有教训
3. 违反教训的候选股：
   - 扣减对应维度 1-2 分
   - 如果违反多条，直接排除
4. 在最终输出中标注"已参考历史教训 #{编号}"
